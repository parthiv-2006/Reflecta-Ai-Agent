# eval — reflecta eval harness

This directory contains the evaluation harness that benchmarks reflecta against
a set of known fixture repositories. It lets you detect coverage regressions
before merging a prompt change, compare routing strategies objectively, and
validate that the testability triage is neither over-blocking nor under-blocking.

---

## Quick start

```bash
# 1. Warm the cache (live LLM calls — do once)
python -m reflecta eval cache

# 2. Run the harness (cache hit — zero quota)
python -m reflecta eval run

# 3. Single fixture
python -m reflecta eval run --fixture calc
```

---

## Fixtures

| Fixture       | Source file        | Covered / Total | Purpose                                      |
|---------------|--------------------|-----------------|----------------------------------------------|
| `calc/`       | `calc.py`          | 3 / 6           | Easy tier — pure arithmetic, clean targets   |
| `text_utils/` | `text_utils.py`    | 2 / 5           | Medium tier — regex + conditionals           |
| `risky_io/`   | `risky_io.py`      | 0 / 4           | Triage validation — all targets must be blocked |

Each fixture directory contains:

```
<fixture>/
├── <fixture>.py              # The Python module to test
├── tests/
│   └── test_<fixture>_partial.py   # Pre-existing partial tests (committed)
└── coverage_baseline.json    # Coverage snapshot before reflecta runs (committed)
```

## CI-safe cache workflow

reflecta uses a sha256-keyed disk cache (`llm/cache.py`, 7-day TTL) so that
LLM responses from a previous run can be replayed without spending quota.

### First-time setup (developer machine)

```bash
python -m reflecta eval cache
```

This runs reflecta with **real LLM calls** against every fixture and writes the
responses to `eval/recordings/<fixture>/`. Commit the recordings:

```bash
git add eval/recordings/
git commit -m "eval: commit cache recordings for CI"
```

### Normal operation (CI + dev after first setup)

```bash
python -m reflecta eval run
```

The harness passes `--cache-dir eval/recordings/<fixture>/` to every
`reflecta run` call. All LLM responses are served from the committed cache —
**zero quota is spent**.

### After a prompt change

1. Re-warm the cache for the affected fixtures:
   ```bash
   python -m reflecta eval cache --fixture calc
   ```
2. Inspect the new metrics:
   ```bash
   python -m reflecta eval run --fixture calc
   ```
3. If the regression is intentional (e.g., a tighter prompt that costs one more
   repair), update the baseline:
   ```bash
   python -m reflecta eval update-baseline --fixture calc
   ```
4. Commit the new recordings and baseline together:
   ```bash
   git add eval/recordings/calc/ eval/baselines/baseline.json
   git commit -m "eval: refresh calc cache and baseline after prompt change"
   ```

> **Treat `eval/recordings/` like snapshot tests.** They are the authoritative
> record of what the LLM returned for a given prompt. When prompts change,
> the recordings change too — that is intentional and expected.

## Baseline format

`eval/baselines/baseline.json` maps fixture names to metric constraint dicts:

```json
{
  "calc": {
    "coverage_delta":      { "min": 0.18 },
    "tests_accepted":      { "min": 2 },
    "llm_calls_gemini":    { "min": 1, "max": 6 },
    "targets_skipped_risky": { "exact": 0 }
  },
  "risky_io": {
    "tests_accepted":      { "exact": 0 },
    "llm_calls_gemini":    { "exact": 0 }
  }
}
```

Supported constraint shapes:

| Shape              | Meaning                    |
|--------------------|----------------------------|
| `{ "exact": N }`   | actual == N (deterministic) |
| `{ "min": N }`     | actual >= N                |
| `{ "max": N }`     | actual <= N                |
| `{ "min": lo, "max": hi }` | lo <= actual <= hi |

Tolerances are intentionally **loose on LLM call counts** (stochastic retries)
and **tight on triage outcomes** (deterministic AST analysis).

## Adding a new fixture

1. Create `eval/fixtures/<name>/`:
   ```
   <name>/
   ├── <name>.py           # Source module with known coverage gaps
   ├── tests/
   │   └── test_<name>_partial.py
   └── coverage_baseline.json   # generated by coverage run + coverage json
   ```
2. Generate the baseline:
   ```bash
   cd eval/fixtures/<name>
   python -m coverage run --source=<name> -m pytest tests/ -q
   python -m coverage json -o coverage_baseline.json
   ```
3. Add a baseline entry to `eval/baselines/baseline.json`.
4. Warm the cache: `python -m reflecta eval cache --fixture <name>`.
5. Commit everything.

## What the metrics mean

| Metric                       | Description                                              |
|------------------------------|----------------------------------------------------------|
| `coverage_before/after`      | % coverage before/after the harness run                 |
| `coverage_delta`             | coverage_after − coverage_before                        |
| `targets_attempted`          | Functions where reflecta called the LLM                 |
| `tests_accepted`             | Tests that passed both assertion gate + delta gate      |
| `tests_discarded`            | Tests that were generated but didn't raise coverage     |
| `repair_attempts_used`       | Total Groq repair calls across all targets              |
| `targets_skipped_risky`      | Targets classified RISKY by static triage               |
| `targets_skipped_blocked`    | Targets classified BLOCKED (module can't be imported)   |
| `llm_calls_gemini`           | Gemini Flash generation calls                           |
| `llm_calls_groq`             | Groq repair calls (8B + 70B combined)                   |
| `llm_calls_claude`           | Claude Haiku overflow/escalation calls                  |
